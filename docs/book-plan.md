# 書籍企画書 (Book Plan)

## 書籍概要

### タイトル
**WebAssembly 実践ガイド** ― ブラウザからKubernetesまで

### 形態
- 判型: B5
- 想定ページ数: 50ページ
- 配布形態: 技術書典

### コンセプト
- **ポータブルバイトコードとしての本質理解**: WASMを「ブラウザ用技術」ではなく「ポータブルなバイトコード」として捉え、その本質と可能性を解説する
- **サンプルアプリ駆動**: 画像フィルタWebアプリを全章通じて段階的に構築し、概念理解から本番デプロイまでを体験する
- **判断力の養成**: 「WASMを使うべきか」の判断基準を、実装を通じて読者自身が獲得できるようにする

### ビジョン
本書は、画像フィルタWebアプリを段階的に構築しながら、WebAssemblyの本質を「ポータブルなバイトコード」として理解する実践ガイドである。Rust を主要言語としつつ Go・Python との比較も交え、ブラウザからKubernetesまでの実行環境で活用する力を養う。

### サンプルアプリケーション
**WASM Image Filter** ― ブラウザ上で画像にフィルタ（グレースケール、セピア、ぼかし等）を適用するWebアプリ。画像処理ロジックをRustでWASMモジュールとして実装し、JavaScriptのUIから呼び出す構成。各章で段階的に機能を拡張し、最終的にKubernetesへデプロイする。

## 対象読者

### プライマリーペルソナ
- バックエンド開発者・インフラエンジニア
- Python または Go での開発経験あり（1年以上）
- Kubernetes の基本操作（Pod、Deployment）を理解している
- WASMに興味はあるが、ブラウザ以外での活用方法がわからない

### 前提知識
読者が本書を読む前に理解していることを前提とする:
- いずれかのプログラミング言語での開発経験
- コマンドラインの基本操作
- コンテナ（Docker）の基本概念

### 前提としない知識
本書で説明する:
- WebAssembly の仕様・仕組み
- Rust の基本文法（WASM開発に必要な範囲）
- wasm-pack 等のツールチェーン
- WASI（WebAssembly System Interface）

## 到達目標

読了後、読者が以下をできるようになること:

1. WebAssembly の仕組み（バイナリフォーマット、線形メモリ、モジュール構造）を説明できる
2. Rust で WASM モジュールを作成し、JavaScript から呼び出せる
3. JavaScript と WASM 間のメモリ共有・型変換を実装できる
4. CPU集約的タスクにおける WASM の性能特性を理解し、適用判断ができる
5. Kubernetes 上で WASM ワークロードをデプロイできる

## 全体構成

### 章構成

| 章 | タイトル | ページ（目安） | 性質 |
|----|---------|--------------|------|
| 第1章 | WASMの正体 | 8p | 概念・背景 |
| 第2章 | 最小実装から理解する | 10p | ハンズオン |
| 第3章 | JavaScriptとの連携 | 12p | ハンズオン |
| 第4章 | 実用ユースケース | 10p | 応用・実践 |
| 第5章 | 本番環境への道 | 10p | 運用・デプロイ |

### 各章の概要

**第1章 WASMの正体（8p）**
WASMのバイナリフォーマットとしての本質、JavaScriptの限界とWASMの登場背景、Docker/Kubernetes時代における位置づけを解説する。サンプルアプリの完成イメージを提示し、全体の見通しを示す。

**第2章 最小実装から理解する（10p）**
Rust + wasm-pack でサンプルアプリの基礎を構築する。最小限のWASMモジュール（グレースケール変換）を作成し、ブラウザで動作させる。TinyGo・Pyodide との簡易比較も行う。

**第3章 JavaScriptとの連携（12p）**
サンプルアプリにフィルタ機能（セピア、ぼかし）を追加しながら、WASM-JS間の型変換、メモリ共有（ArrayBuffer）、エラーハンドリングを学ぶ。

**第4章 実用ユースケース（10p）**
サンプルアプリのパフォーマンスをJavaScript実装と比較し、WASMの性能特性を検証する。バッチ処理（複数画像の一括変換）を追加し、CPU集約的タスクでの優位性を示す。

**第5章 本番環境への道（10p）**
サンプルアプリのバンドルサイズ最適化、ブラウザ互換性対応を行い、最終的にKubernetes上（containerd + runwasi）にデプロイする。

## 技術環境

- **主要な技術スタック**: wasm-pack, wasm-bindgen, wasmtime, TinyGo, Pyodide, Kubernetes (containerd + runwasi)
- **コード例の言語**: Rust（メイン）、Go、Python、JavaScript

## 執筆方針

- **サンプルアプリ駆動**: 画像フィルタアプリを軸に、各章で機能を追加しながら概念を学ぶ構成とする
- **コード中心・説明最小限**: 概念説明は必要最小限に留め、サンプルアプリのコードで理解を促進する
- **判断基準の提示**: 各ユースケースで「WASMを使うべき場面」と「使わなくてよい場面」を明示する

## スコープ外

明示的にスコープ外とする項目:
- Rust 言語の体系的な解説（WASM開発に必要な範囲のみ扱う）
- ブラウザ拡張・PWA との統合
- WASM を使ったゲーム開発
- AssemblyScript 等の WASM 専用言語
- 科学計算・リアルタイムメディア処理の詳細（概要のみ言及）

## 成功指標

- 読者が Rust で WASM モジュールを作成し、ブラウザで動作させられる
- 読者が自身のプロジェクトで「WASMを使うべきか」を判断できる
- Kubernetes 上での WASM デプロイ手順を再現できる
