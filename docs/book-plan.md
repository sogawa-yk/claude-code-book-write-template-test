# 書籍企画書 (Book Plan)

## 書籍概要

### タイトル
**WebAssembly 実践ガイド** ― ブラウザからKubernetesまで

### 形態
- 判型: B5
- 想定ページ数: 50ページ
- 配布形態: 技術書典

### コンセプト
- **ポータブルバイトコードの本質理解**: WASMを「ブラウザ用技術」ではなく「ポータブルなバイトコード」として捉え、その本質と可能性を解説する
- **実装と運用の両面カバー**: コードを書くだけでなく、本番環境での運用知識まで一貫して扱う
- **判断力の習得**: 「WASMを使うべきか否か」を自分で判断できる力を養う

### ビジョン
WebAssemblyはブラウザ向け技術として登場したが、現在はサーバーサイドやエッジコンピューティングにも活用が広がっている。本書は、Python/Go経験者やKubernetes運用者を対象に、WASMの本質を最小限のコードで理解し、ブラウザからKubernetesまでの実行環境を横断的に体験する。50ページというコンパクトな分量で、WASMの「何がうれしいのか」「どこで使うべきか」を判断できる実践力を身につけることを目指す。

## 対象読者

### プライマリーペルソナ
- Web開発者またはインフラエンジニア
- Python/Goでの開発経験がある（中級以上）
- Kubernetes の基本的な運用経験がある
- WASMに興味があるが、実際に触ったことがない、または入門レベルで止まっている

### 前提知識
読者が本書を読む前に理解していることを前提とする:
- Python または Go の基本文法と開発経験
- HTML/JavaScript の基礎知識
- Docker コンテナの基本概念
- コマンドラインツールの操作

### 前提としない知識
本書で説明する:
- WebAssembly の仕組みと仕様
- Rust の基礎（WASM実装に必要な範囲のみ）
- wasm-pack, wasmtime 等のツールチェーン
- WASI（WebAssembly System Interface）の概念

## 到達目標

読了後、読者が以下をできるようになること:

1. WASMのアーキテクチャと動作原理を説明できる
2. Rust + wasm-pack で基本的なWASMモジュールを作成できる
3. JavaScriptからWASM関数を呼び出し、メモリ共有の仕組みを理解している
4. WASMが有効なユースケース（CPU集約処理等）を判断できる
5. KubernetesでWASMワークロードを実行する方法を理解している

## 技術環境

- **主要な技術スタック**: WebAssembly, WASI, wasm-pack, wasm-bindgen, wasmtime, Kubernetes (containerd + runwasi)
- **コード例の言語**: Rust（メイン）、JavaScript、Go（比較用）、Python（比較用）

## 執筆方針

- **コンパクトさ優先**: 50ページの制約を活かし、各トピックの本質だけを凝縮して伝える。冗長な説明は排除する
- **手を動かす構成**: 各章に実行可能なコード例を含め、読者が手元で試せるようにする
- **判断材料の提供**: 「WASMを使うべき場面」と「使わなくてよい場面」を明確に示し、読者の意思決定を支援する

## スコープ外

明示的にスコープ外とする項目:
- Rust言語自体の詳細な解説（必要最小限に留める）
- WASMの仕様書レベルの詳細（バイナリフォーマットの内部構造等）
- ブラウザごとの互換性対応の網羅的な解説
- 科学計算やリアルタイムメディア処理の詳細実装（概要のみ言及）
- AssemblyScript、Emscripten等の代替ツールチェーンの詳細

## 成功指標

- 読者がWASMモジュールを自力でビルド・実行できる
- 「WASMを採用すべきか」の判断基準を自分の言葉で説明できる
- KubernetesでのWASMワークロード実行を理解し、検証環境で試せる
