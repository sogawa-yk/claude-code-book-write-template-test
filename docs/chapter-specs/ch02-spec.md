# 第2章 最小実装から理解する ― 仕様書

## 章の概要

**章の目的**: Rust + wasm-packでWASMモジュールを作成し、ブラウザで動作させる一連のフローを体験する。サンプルアプリの基礎（グレースケール変換）を構築する。

**目標ページ数**: 10p

## 前提知識

この章を読むために必要な前提:
- 第1章の内容（WASMの基本概念）
- コマンドライン操作の基礎

### 2.3 サンプルアプリ: グレースケール変換

**概要**: サンプルアプリの第一段階として、画像のグレースケール変換をRustで実装する。ピクセルデータの受け渡しを通じて、WASMの基本的なデータ処理パターンを学ぶ。

**図表**:
- 図2.3: グレースケール変換のデータフロー

**キーポイント**:
- ピクセルデータ（RGBA）のバイト配列としての扱い
- `&[u8]` と `Vec<u8>` によるデータの受け渡し

**サンプルアプリ要件**:
- Rustで `grayscale()` 関数を実装
- wasm-packでビルドし、HTMLページから呼び出して動作確認

### 2.4 言語比較: TinyGoとPyodide

**概要**: 同じグレースケール変換をTinyGoとPyodideで実装し、バイナリサイズ・パフォーマンスを簡易比較する。

**図表**:
- 表2.1: 言語別WASMビルド比較（バイナリサイズ、ビルド時間、エコシステム成熟度）

**キーポイント**:
- Rustが最もバイナリサイズが小さく、エコシステムも成熟している
- 用途に応じた言語選択の基準
