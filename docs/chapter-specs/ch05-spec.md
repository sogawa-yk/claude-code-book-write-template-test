# 第5章 本番環境への道 ― 最適化とKubernetesデプロイ ― 仕様書

## 章の概要

**章の目的**: WASMバイナリの最適化手法を理解し、KubernetesでWASMワークロードを実行する方法を習得する。本番運用に必要な知識を身につける。

**目標ページ数**: 14p

**所属する部**: 第IV部 本番環境への道

## 前提知識

この章を読むために必要な前提:
- 第2章の内容（wasm-packによるビルド）
- 第4章の内容（WASMの実用パターンと採用判断）
- Dockerコンテナの基本概念
- Kubernetesの基本操作（kubectl）

## 節の構成

### 5.1 バンドルサイズ最適化

**概要**: WASMバイナリのサイズを削減するための手法を解説する。Cargo.tomlの設定、wasm-opt、ツリーシェイキング等の具体的な最適化手法を示す。

- 不要なクレートの削除とツリーシェイキング

### 5.2 ブラウザ互換性とロード戦略

**概要**: WASMのブラウザサポート状況を整理し、本番環境でのロード戦略（遅延ロード、ストリーミングコンパイル）を解説する。

**図表**:

**キーポイント**:
- `WebAssembly.instantiateStreaming`によるストリーミングコンパイル
- Code Splitting と遅延ロードのパターン
- Content-Type（application/wasm）の設定

### 5.3 パフォーマンス分析

**概要**: WASMアプリケーションのパフォーマンスを計測・分析する手法を解説する。ブラウザのDevToolsを活用したプロファイリング方法を示す。

**図表**:

**キーポイント**:

- メモリ使用量の監視

### 5.4 KubernetesでのWASM実行

**概要**: containerdとrunwasiを使い、Kubernetes上でWASMワークロードを実行する方法を解説する。コンテナとWASMの比較、デプロイ手順を示す。

**図表**:

- 起動時間とリソース効率の優位性
