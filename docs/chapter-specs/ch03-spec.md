# 第3章 JavaScriptとの連携 ― 型変換とメモリ共有 ― 仕様書

## 章の概要

**章の目的**: WASMモジュールとJavaScriptの間でデータをやり取りする仕組みを理解し、画像フィルタ処理を例に実践的な連携パターンを習得する。

**目標ページ数**: 8p

**所属する部**: 第III部 実用パターン

## 前提知識

この章を読むために必要な前提:
- 第2章の内容（wasm-packによるビルド、wasm-bindgenの基礎）
- JavaScriptの基礎知識（関数、配列、非同期処理）

## 節の構成

### 3.1 WASM関数の呼び出しパターン

**概要**: JavaScriptからWASM関数を呼び出す基本パターンを解説する。同期呼び出し、引数と戻り値の受け渡し、エラーハンドリングを扱う。

**図表**:
- 図3.1: JS-WASM間の関数呼び出しフロー

### 3.2 メモリ共有の仕組み ― ArrayBufferと線形メモリ

**概要**: WASMの線形メモリとJavaScriptのArrayBufferの関係を解説する。バイナリデータの効率的な受け渡し方法を示す。

**図表**:
- 図3.2: WASMの線形メモリとArrayBufferの関係

**キーポイント**:
- 線形メモリはArrayBufferとして公開される
- ポインタとサイズによるデータの受け渡し
- メモリの成長（grow）とビューの無効化に注意

### 3.3 実例：画像フィルタ処理

**概要**: Rustで画像のグレースケール変換フィルタを実装し、JavaScript側でCanvas APIと組み合わせてブラウザ上で動作させる。一連の実装を通じて、バイナリデータの受け渡しを実践する。

**図表**:
- 図3.3: 画像フィルタ処理のデータフロー ― Canvas → WASM → Canvas

**キーポイント**:
- Canvas APIからのピクセルデータ取得
- WASMへのバイナリデータ転送と処理
- 処理結果のCanvas への反映
