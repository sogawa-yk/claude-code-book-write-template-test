# 第3章 JavaScriptとの連携 ― 仕様書

## 章の概要

**章の目的**: JavaScript-WASM間のメモリ共有、型変換、エラーハンドリングを理解し、サンプルアプリにセピア・ぼかしフィルタを追加する。

**目標ページ数**: 12p

## 前提知識

この章を読むために必要な前提:
- 第2章の内容（wasm-packでのビルド、基本的なWASM関数の作成）
- JavaScriptの基本（DOM操作、Canvas API）

## 節の構成

### 3.1 メモリ共有の仕組み

**概要**: WASMの線形メモリとJavaScriptのArrayBufferの関係を解説する。データコピーと共有メモリの違い、パフォーマンスへの影響を説明する。

**図表**:
- 図3.1: WASM線形メモリとArrayBufferの関係

**キーポイント**:
- WASMの線形メモリはJSからArrayBufferとしてアクセス可能
- データコピーを避けることでパフォーマンスが向上する

### 3.2 型変換とデータの受け渡し

**概要**: WASMの数値型（i32, f64等）とJavaScriptの型の対応、文字列・配列の受け渡しパターンを解説する。wasm-bindgenが自動生成するバインディングの仕組みにも触れる。

**図表**:
- 表3.1: WASM型とJavaScript型の対応表
- 図3.2: wasm-bindgenによるバインディング生成フロー

**キーポイント**:
- 数値型は直接渡せるが、複合型はシリアライズが必要
- wasm-bindgenが型変換の大部分を自動化する

### 3.3 サンプルアプリ: セピア・ぼかしフィルタの追加

**概要**: サンプルアプリにセピアフィルタとぼかしフィルタ（ボックスブラー）を追加する。Canvas APIを使ったプレビュー表示も実装する。

**図表**:
- 図3.3: フィルタ処理パイプライン（画像読み込み→フィルタ選択→WASM処理→Canvas描画）

**キーポイント**:
- 画像データ（RGBA配列）の効率的な受け渡し
- Canvas `getImageData()` / `putImageData()` との連携

**サンプルアプリ要件**:
- `sepia()` 関数と `blur()` 関数をRustで実装
- フィルタ選択UI（ドロップダウン）を追加
- Canvas上でリアルタイムプレビュー

### 3.4 エラーハンドリング

**概要**: WASMモジュール内でのパニック処理、JavaScript側でのエラーキャッチ、`console_error_panic_hook` の導入を解説する。

**図表**:
- 図3.4: WASM-JS間のエラー伝播フロー

**キーポイント**:
- WASMのパニックはデフォルトでは不親切なエラーメッセージになる
- `console_error_panic_hook` で詳細なスタックトレースを取得できる

## 章末の理解度チェック問題

### 問題の方針
- 問題数: 4問
- 問題の種類: 概念の確認、判断問題、設計問題

### 問題案
1. WASMの線形メモリとJavaScriptのArrayBufferはどのような関係か（概念の確認）
2. 文字列をWASMに渡す場合、内部で何が起きているか（概念の確認）
3. 大量のデータをWASMに渡す場合、コピーと共有メモリのどちらを選ぶべきか（判断問題）
4. 新しいフィルタ（例: エッジ検出）を追加する場合の設計方針（設計問題）
