# 第4章 実用ユースケース ― CPU集約処理とバイナリ処理 ― 仕様書

## 章の概要

**章の目的**: WASMが特に有効なユースケース（CPU集約処理、バイナリデータ処理）を実装し、「WASMを使うべき場面」と「使わなくてよい場面」の判断力を養う。

**目標ページ数**: 8p

**所属する部**: 第III部 実用パターン

## 前提知識

この章を読むために必要な前提:
- 第2章の内容（Rust + wasm-packによるビルド）
- 第3章の内容（JS-WASM間のデータ受け渡し）

## 節の構成

### 4.1 CPU集約処理 ― 素数判定とベンチマーク

**概要**: 素数判定を題材に、JavaScriptとWASMの性能差をベンチマークで比較する。WASMが有効な「CPU集約処理」の特徴を整理する。

**図表**:
- 図4.1: JavaScript vs WASM ― CPU集約処理のベンチマーク結果
- 表4.1: WASMが有効なユースケースの分類

**キーポイント**:
- CPU集約処理でのWASMの性能優位性
- ベンチマークの正しい取り方
- I/O待ちが多い処理ではWASMの優位性は小さい

### 4.2 バイナリデータ処理 ― 圧縮・解凍

**概要**: RustのflateライブラリをWASMにコンパイルし、ブラウザ上でのデータ圧縮・解凍を実装する。既存のRustエコシステムをWASMで活用するパターンを示す。

**図表**:
- 図4.2: バイナリ処理のアーキテクチャ ― Rustライブラリの再利用パターン

**キーポイント**:
- 既存のRustクレートをWASMで再利用できる
- バイナリデータの効率的な受け渡し（線形メモリ活用）
- ブラウザのネイティブAPIとの使い分け

### 4.3 判断フレームワーク ― WASMを使うべきか

**概要**: これまでの実装経験を踏まえ、WASMの採用判断フレームワークを提示する。適するケースと適さないケースを整理し、意思決定のガイドラインを提供する。

**図表**:
- 図4.3: WASM採用判断フローチャート

**キーポイント**:
- WASMが有効な条件（CPU集約、バイナリ処理、既存ライブラリ活用）
- WASMが不要な条件（DOM操作中心、I/O中心、小規模処理）
- 導入コスト（ツールチェーン、学習コスト）も判断に含める
