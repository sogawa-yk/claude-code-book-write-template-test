**章の目的**: WASMの性能特性をJavaScript実装との比較で定量的に理解し、バッチ処理の実装を通じてCPU集約的タスクでの適用判断基準を獲得する。

**目標ページ数**: 10p

## 節の構成

### 4.1 WASM vs JavaScript ― 性能比較

**概要**: サンプルアプリのグレースケール変換を題材に、同等のJavaScript実装と処理時間を比較する。`performance.now()` を使った計測手法も示す。

**図表**:
- 図4.1: WASM vs JS処理時間の比較グラフ（画像サイズ別）
- 表4.1: ベンチマーク結果一覧

**キーポイント**:
- 画像サイズが大きいほどWASMの優位性が顕著になる
- 小規模データではWASM呼び出しのオーバーヘッドが支配的

### 4.2 サンプルアプリ: バッチ処理の実装

**概要**: 複数画像の一括フィルタ適用機能を追加する。バッチ処理におけるメモリ管理の工夫（事前確保、再利用）を解説する。

**図表**:
- 図4.2: バッチ処理のデータフロー

**キーポイント**:
- メモリの事前確保でアロケーションコストを削減
- バッチサイズとメモリ使用量のトレードオフ

**サンプルアプリ要件**:
- `batch_apply()` 関数をRustで実装
- 複数ファイルの選択UI追加
- 処理時間の表示機能

### 4.3 WASMを使うべきとき、使わなくてよいとき

**概要**: ここまでの実装経験を元に、WASMの適用判断フレームワークを提示する。CPU集約的タスク、I/Oバウンドタスク、DOM操作等の観点から整理する。

**図表**:
- 図4.3: WASM適用判断フローチャート
- 表4.2: ユースケース別の推奨度

**キーポイント**:
- CPU集約的タスク → WASM有利
- DOM操作・I/Oバウンド → JavaScript有利
- 判断基準はデータ量と計算量
