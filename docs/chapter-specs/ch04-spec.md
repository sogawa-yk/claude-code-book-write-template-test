# 第4章 データ構造 ― 仕様書

## 章の概要

**章の目的**: リスト、タプル、辞書の特徴と操作方法を理解し、適切なデータ構造を選択して使えるようになること。

**目標ページ数**: 4p

## 前提知識

この章を読むために必要な前提:
- 第2章の内容（変数、データ型の理解）
- 第3章の内容（forループによる反復処理）

## 節の構成

### 4.1 リスト

**概要**: リストの作成、要素のアクセス、追加・削除・変更の操作を説明する。インデックスとスライスの使い方を扱う。

**図表**:
- 図4.1: リストの構造とインデックスの対応

**キーポイント**:
- リストは`[]`で作成し、順序を持つ変更可能なコレクションである
- インデックスは0から始まり、負のインデックスで末尾からアクセスできる
- `append()`, `remove()`, `len()`等のメソッドで操作する

### 4.2 タプルと辞書

**概要**: タプルの特徴（変更不可能）と辞書のキー・バリュー構造を説明する。リストとの違いと使い分けを示す。

**図表**:
- 表4.1: リスト・タプル・辞書の比較表

**キーポイント**:
- タプルは`()`で作成し、変更不可能である
- 辞書は`{}`でキーと値のペアを管理する
- データの性質に応じて適切なデータ構造を選択する

### 4.3 データ構造の操作

**概要**: リスト内包表記、辞書の反復処理、ネストしたデータ構造の操作を紹介する。実践的な使用例を示す。

**図表**:
- 図4.2: ネストしたデータ構造のイメージ

**キーポイント**:
- リスト内包表記は`[式 for 変数 in イテラブル]`で簡潔にリストを生成できる
- `items()`で辞書のキーと値を同時に取得できる
- データ構造はネストして複雑なデータを表現できる

- 問題の種類:
